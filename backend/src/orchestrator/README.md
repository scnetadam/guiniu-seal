# 龟钮协调层 — Orchestrator

跨项目协调引擎，负责三项目（印信/印证/自驭）的工作流串联。

## 核心职责

| 流 | 来源 | 目标 | 说明 |
|----|------|------|------|
| 支付存证 | 印信 settle | 印证 notary | 每笔结算自动创建存证 |
| 权重协调 | 自驭 content | 印信 settle | 内容访问权重 → 支付分账 |
| 税务闭环 | 印信 settle | 自驭 track | 税务轨状态同步 |
| 文档生成 | 三项目 | docs/ | 运营/合规/税务报告 |
| 收益通知 | 印信 settle | 用户 | 短信/邮件/站内信 |
| 多平台同步 | 所有 | GitHub等 | 开源数据追踪 |

## 架构

```
┌─────────────┐     ┌──────────────┐     ┌─────────────┐
│  印信 seal  │◄────┤  Orchestrator ├────►│  印证 verify │
│  支付/结算  │     │  协调层       │     │  存证/公证   │
└──────┬──────┘     └──────┬───────┘     └──────┬──────┘
       │                   │                     │
       │                   ▼                     │
       │          ┌──────────────┐               │
       └─────────►│  自驭 deveco │◄──────────────┘
                  │  AI/权重/内容 │
                  └──────────────┘
```

## 冗余机制

- 协调操作写 registry 日志，失败自动重试（3次）
- 协调队列持久化，重启后可追赶未完成操作
- 关键操作支持手动回放
